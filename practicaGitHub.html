En el ejercicio de temas avanzado 1 , se plantea el problema de que el personal de administracion y que necesitan que los comit esten en un orden secuencial.


El problema esta planteado de esta forma, por lo que se necesita reorganizar las ramas y colocar los commit para que queden de un orden secuencial.

El primer comando que vamos a utilizar es un git rebase, esto permite modificar de una rama a otra. Por lo que la rama bug Fix esta asignada al comit c3, introducimos el comando git rebase main bugFix, para que el comit c3 se integre detras de la rama main que en este caso esta asignada al comit c2.


El siguiente paso es similar al anterior pero en este vamos a utilizar el mismo comando, pero vamos a mover la rama side detras de la rama bug Fix. En la rama side se encontraban los commit c4, c5 y c6 por los que todos estos se colocaron detras del comit c3 y estariamos llevando el orden que nos pide el personal de administracion.


El siguiente paso es modificar la rama another que esta asignada al comit c7 como podemos observar en la segunda imagen. El proceso es el mismo que en los anteriores, utilizando el comando git rebase side another.



Como podemos observar la rama main esta asignada al comit c2, la rama main es una rama predeterminada que se crea automaticamente cuando se crea un repositorio, por lo que utilizando git rebase another main movemos esta rama junto a la rama another.

En el ejercicio de hasta el origen y mas alla 2, se plantea el problema de subir tus archivos a remoto haciendo un push.Explica que hay 2 comandos para hacerlo.


Lo primero de todo es hacer un Git checkout para trabajar sobre la rama main, y posteriormente realizaremos un git pull que se utiliza para extraer y descargar contenido desde un repositorio remoto y actualizar al instante el repositorio local para reflejar ese contenido.


posteriormente vamos a realizar un git merge para unificar la rama side y poder trabajar en ella para realizar un push, primero realizaremos el git merge en el side 1, posteriormente en el side2 y por último en el side3.


Por ultimo realizaremos un git push para cargar el contenido del local al remoto y poder tener acceso a el desde Git Hub.
